# Structured Data Policy

Structured data recommendations must be tied to a URL class and validated against visible page content.

## Required Checks

- Product schema for products: name, image, SKU where appropriate, offers, price, currency, availability, and reviews only when real reviews exist.
- Breadcrumb schema for navigational hierarchy.
- Article schema for editorial content when appropriate.
- Organization/WebSite schema consistency with Yoast output.
- Duplicate schema from theme, plugins, and custom code.
- Rich result eligibility vs guaranteed display; never promise rich results.

## Restrictions

Do not fabricate reviews, ratings, prices, availability, authors, publication dates, or organization details. Concrete schema Findings belong in Runs.
