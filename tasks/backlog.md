# Task Backlog

This file is the current canonical cross-run Task status registry. It remains the canonical owner of Task status until a Schema, Validator, and Generator are implemented and merged in a later PR. Individual Task records remain the canonical owners of approval evidence, implementation history, detailed Validation, rollback details, and monitoring notes. After the future Generator is implemented and merged, individual Task records will own canonical Task status and this file will become a generated view. These phases must not overlap in a way that creates two simultaneous sources of truth. Task IDs use the `TSK-YYYY-NNN` format. `TSK-2026-001` is an example format only, not an assigned identifier.
